2. JSX: "HTML Gibi Ama JavaScript"


En Kritik Konu: Server vs. Client Components
Next.js ile direkt başlandığında en büyük avantaj Server Components mantığını varsayılan olarak öğrenmektir. Eski C/C++ veya Backend mantığına sahip biri bunu çok sevecektir.

Server Component (Varsayılan): Kod sunucuda çalışır, HTML üretilir ve tarayıcıya gönderilir. console.log tarayıcıda değil, terminalde görünür. Veritabanına buradan direkt bağlanılabilir (Güvenlidir).

Client Component ("use client"): Eğer butona tıklama (onClick), useState (durum yönetimi) veya useEffect gibi tarayıcı etkileşimi gerekiyorsa dosyanın en tepesine "use client" yazılmalıdır.

Analoji:

Server Component: PHP/ASP gibi çalışır (Static, güvenli, backend).

Client Component: jQuery/Pure JS gibi çalışır (İnteraktif, tarayıcıda).








Props (Veri Aktarımı) ve Komponentleşme

Props: Fonksiyon parametresi gibidir. Ana sayfadan karta veri göndermek için kullanılır.

TypeScript: Burada TypeScript devreye girer. interface ile hangi verinin geleceğini tanımlamak, C++ geçmişi olan biri için çok rahatlatıcıdır (Tip güvenliği).
